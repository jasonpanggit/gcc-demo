## 2. Critical Integration Defects

### D1 [HIGH]: Priority and domain metadata discarded during registration
**Location:** \`utils/mcp_host.py:266-267\`
**Impact:** Priority-based collision resolution broken; multi-domain YAML \`domains[]\` ignored
**Root Cause:** \`MCPServerConfig\` object not threaded through to \`_register_clients()\`
**Fix Path:** Call \`registry.register_server()\` directly in \`from_config()\` where \`server_cfg\` is in scope
**Estimated Effort:** 2-4 hours

**Details:**
The YAML defines different priorities:
- \`azure\`: \`priority: 5\` (highest)
- Standard servers: \`priority: 10\`
- \`azure_cli_executor\`: \`priority: 15\` (lowest)

However, all servers register with hardcoded \`priority=100\` and \`domain=label\`, making collision resolution first-come-first-served instead of priority-based.

## 3. Medium Defects

### D2: Incorrect retry_async usage example

### D3: Incorrect TryAgain usage example

### D4: No retry on transient factory failures
